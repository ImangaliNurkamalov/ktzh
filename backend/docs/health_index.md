# Индекс здоровья локомотива (формула)

Расчёт выполняется на бэкенде в [`app/services/health_service.py`](../app/services/health_service.py) в функции `compute_health_from_ingress`. Поле `health` с борта не принимается.

Итог: целое число **0–100** (`HealthIngress.index`) и строковый статус **`normal` | `warning` | `critical`**.

---

## 1. Поэлементные оценки \(s_i \in [0, 1]\)

Каждый учитываемый канал телеметрии преобразуется в оценку **\(s_i\)**: **1** — норма, **0** — худший допущенный режим, между ними — линейная интерполяция (где применимо).

### 1.1. Линейная интерполяция `_lin_clamp(x, x0, x1, y0, y1)`

- при \(x \le \min(x_0, x_1)\) возвращается соответствующий конец;
- при \(x \ge \max(x_0, x_1)\) — другой конец;
- между концами — линейно.

Используется для давлений, температур, напряжений и т.д. (конкретные пороги заданы в коде).

### 1.2. Каналы и смысл шкалы (кратко)

| Элемент | Идея шкалы |
|--------|------------|
| `tm_pressure` | выше ~4.5 → 1; ниже ~2.0 → 0; между — линейно |
| `gr_pressure` | выше ~7.0 → 1; ниже ~5.0 → 0; между — линейно |
| `oil_pressure` | выше ~3.0 → 1; ниже ~1.5 → 0; между — линейно |
| `oil_temp` | до ~85 °C → 1; к ~105 °C → 0; между — линейно (рост хуже) |
| `coolant_temp` | до ~82 °C → 1; к ~100 °C → 0; между — линейно |
| `board_voltage` | отклонение от 110 В до ~8 В → 1; к ~18 В и больше → 0; между — линейно |
| `catenary_voltage` | выше ~25 кВ → 1; ниже ~18 кВ → 0; между — линейно |
| `transformer_temp` | до ~78 °C → 1; к ~92 °C → 0; между — линейно |
| `traction_current` | до ~650 А → 1; к ~800 А → 0; между — линейно |
| `fuel_level` | от 20% и выше → 1; к 0% — линейно к 0 |

### 1.3. Дискретные каналы

- **`wheel_slip`**: буксование есть → 0, нет → 1.
- **`pantograph`**: строка `"raised"` (без учёта регистра) → 1, иначе → 0.

### 1.4. Подшипники `bearings_max` (кусочная шкала)

- \(T \le 70\) °C → \(s = 1\)
- \(70 < T \le 80\) °C → плавное снижение от 1 к 0.25
- \(80 < T \le 95\) °C → от 0.25 к 0
- \(T > 95\) °C → \(s = 0\)

### 1.5. Качество датчика (`state`)

Если у канала **`state == 2`** (неисправность по бортовой логике), итоговая оценка по этому каналу ограничивается:

\[
s_i' = \min(s_i,\ 0.42)
\]

Так «хорошие» числа при мёртвом датчике не дают высокого вклада.

---

## 2. Набор элементов и веса

Учитываются только элементы своего типа локомотива:

- **общие** (`common`) — и дизель, и электровоз;
- **только дизель** (`diesel`);
- **только электровоз** (`electric`).

У каждого элемента заданы:

- **вес** \(w_i > 0\) — относительная важность в среднем;
- флаг **`critical`** — участвует ли канал в «полу» по критичным показателям.

Актуальный список (вес, критичность, домен) задан в коде в `_ELEMENT_SPECS`.

---

## 3. «Сырой» индекс в долях единицы

Сначала считается **взвешенное среднее** по всем включённым элементам:

\[
h_{avg} = \frac{\sum_i w_i\, s_i}{\sum_i w_i}
\]

Затем по всем элементам с **`critical = true`** берётся минимум:

\[
c_{floor} = \min\{\, s_i \mid i \text{ критичный} \,\}
\]

(если критичных нет — в коде используется 1.0, на практике критичные всегда есть).

**Итог до сглаживания:**

\[
h_{raw} = \min(h_{avg},\ c_{floor})
\]

Смысл: если хотя бы один критичный канал в нуле (ТМ, буксование, пантограф и т.д.), **нельзя «усреднить»** хорошими остальными датчиками — индекс ограничивается снизу этим минимумом.

---

## 4. Сглаживание по времени (по `locomotive_id`)

Для каждого `locomotive_id` хранится своё сглаженное значение \(h_{smooth}\). На каждом тике:

1. \(h_{raw}\) ограничивается отрезком \([0, 1]\).
2. Первый приход для локомотива: \(h_{smooth} \leftarrow h_{raw}\).
3. Далее — экспоненциальное сглаживание:

\[
h_{smooth} \leftarrow (1 - \alpha)\, h_{smooth} + \alpha\, h_{raw}
\]

где

- если \(h_{raw} < h_{smooth}\) (ухудшение): \(\alpha = \alpha_{down}\) (в коде по умолчанию **0.38**);
- если \(h_{raw} \ge h_{smooth}\) (улучшение или стабильность): \(\alpha = \alpha_{up}\) (по умолчанию **0.09**).

Реакция на аварию быстрее, чем «отпускание» индекса вверх после восстановления телеметрии.

---

## 5. Индекс 0–100 и статус

\[
\text{index} = \mathrm{round}\bigl(100 \cdot \mathrm{clamp}_{[0,1]}(h_{smooth})\bigr)
\]

Пороги статуса:

| Условие | `status` |
|---------|----------|
| index ≥ 85 | `normal` |
| 60 ≤ index < 85 | `warning` |
| index < 60 | `critical` |

---

## 6. Пример (обрыв магистрали ТМ)

Пока ТМ в норме: все \(s_i \approx 1\) для некритичных и критичных → \(h_{avg} \approx 1\), \(c_{floor} = 1\) → \(h_{raw} \approx 1\) → после сглаживания index около **100**, статус **normal**.

При резком падении ТМ оценка \(s_{tm}\) падает; как только критичный минимум становится низким, \(h_{raw} = \min(h_{avg}, c_{floor})\) **следует за худшим критичным каналом**. Сглаживание с большим \(\alpha_{down}\) переводит отображаемый index в **warning**, затем в **critical** за несколько секунд (тиков), даже если остальные подсистемы формально в порядке.
